# Java 后端学习笔记 01：Java 基础与面向对象（详细版）

## 1. 学习目标与课程定位
本章面向 Java 后端初学者，目标是让你建立完整的语言基础与对象建模能力。学完后，你应当能够：
- 独立阅读并编写中小规模 Java 业务代码。
- 正确理解值传递、对象引用、作用域、生命周期等基础概念。
- 使用面向对象思想对业务场景建模，并写出可扩展的类结构。
- 为后续学习集合、并发、Spring、数据库访问打下可靠基础。

## 2. Java 程序结构与执行机制
### 2.1 Java 文件与类
- 一个 `.java` 文件中可以有多个类，但最多一个 `public` 类，且文件名必须与该 `public` 类同名。
- `main` 方法是传统 Java 应用入口：`public static void main(String[] args)`。

### 2.2 编译与运行
- 编译：`javac Hello.java`，生成 `Hello.class`。
- 运行：`java Hello`。
- 本质：Java 源码先编译为字节码，再由 JVM 执行，具备跨平台能力。

### 2.3 JVM / JRE / JDK
- JVM：Java 虚拟机，负责执行字节码。
- JRE：运行环境（JVM + 核心类库）。
- JDK：开发工具包（JRE + 编译器等工具）。

## 3. 数据类型、变量与运算
### 3.1 基本数据类型
- 整数：`byte`、`short`、`int`、`long`
- 浮点：`float`、`double`
- 字符：`char`
- 布尔：`boolean`

### 3.2 引用类型
- 类、接口、数组、枚举、注解都属于引用类型。
- 引用变量存储的是对象地址（引用值），不是对象本体。

### 3.3 自动类型提升与转换
- 小类型参与运算会先提升为 `int`。
- 强制类型转换可能丢失精度，需要显式转换。

### 3.4 常见坑
- `long` 字面量要加 `L`。
- `float` 字面量要加 `f`。
- `double` 金额计算有精度问题，金融场景用 `BigDecimal`。

## 4. 流程控制与方法
### 4.1 条件分支
- `if/else` 适合范围判断。
- `switch` 适合离散值判断。

### 4.2 循环结构
- `for`：次数明确。
- `while`：条件驱动。
- `do...while`：至少执行一次。

### 4.3 方法设计
- 方法参数是值传递。
- 避免“上帝方法”，单个方法职责要清晰。
- 方法命名使用动词短语，例如 `calculateScore`、`buildCoursePlan`。

## 5. 面向对象三大特性
### 5.1 封装
- 字段私有化，暴露必要行为。
- 对外提供稳定接口，隐藏内部实现细节。

### 5.2 继承
- 子类复用父类行为，支持方法重写。
- 继承表达 is-a 关系，例如 `Student extends User`。

### 5.3 多态
- 运行期动态分派，提升扩展性。
- 常用于“面向接口编程”与“策略模式”。

## 6. 抽象类与接口的工程化选择
### 6.1 抽象类适用场景
- 多个子类共享状态或通用实现。
- 希望在父类中提供模板方法。

### 6.2 接口适用场景
- 表达能力契约，强调可替换性。
- 在 Spring 项目中，通常 service 层优先定义接口。

### 6.3 组合优于继承
- 对复杂业务更推荐“对象组合”，降低层级耦合。

## 7. equals / hashCode / toString
### 7.1 为什么重要
- `HashMap`、`HashSet` 等依赖 `hashCode` 和 `equals` 协作。
- 覆写 `equals` 时通常必须覆写 `hashCode`。

### 7.2 常见错误
- 使用 `==` 比较字符串内容。
- 实体对象放入 `HashSet` 后再修改参与 hash 的字段。

## 8. String、StringBuilder 与不可变性
- `String` 不可变，线程安全，适合常量与只读文本。
- 频繁拼接字符串应使用 `StringBuilder`。
- 高并发日志拼接注意对象创建成本。

## 9. 异常体系与基础规范
### 9.1 异常分类
- `Checked Exception`：编译期检查异常。
- `Unchecked Exception`：运行时异常，继承 `RuntimeException`。

### 9.2 基础实践
- 不要吞异常：至少记录日志并附带上下文。
- 对外接口返回可理解错误码，不暴露内部栈信息。

## 10. 课程助教系统中的 OOP 建模示例
### 10.1 业务对象
- `Student`：学生信息与学习画像。
- `Course`：课程元信息。
- `QuestionRecord`：提问记录与答案。
- `KnowledgeChunk`：知识片段及向量索引。

### 10.2 行为抽象
- `AnswerStrategy`：答疑策略接口。
- `RiskFilter`：风险内容过滤接口。
- `MemoryStore`：记忆存储接口（可由 Redis/MySQL 实现）。

## 11. 常见代码坏味道
- 过度使用 `static`，导致状态难管理、难测试。
- 巨型实体类同时承担数据、校验、业务逻辑。
- 方法参数过多，建议封装参数对象。
